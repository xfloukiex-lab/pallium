The problem

Your AI lives in someone else's cloud.

The fediverse gave people sovereign social media and self-hosted storage. The newest and most invasive layer — AI — still runs inside a handful of Big-Tech clouds. Every prompt, document, and memory you give an assistant leaves your control.

Pallium closes that gap — a single, easy-to-deploy package that runs a complete private AI on a machine you own.

The stack

Seven composable pieces. One command to run.

Deployed declaratively with NixOS — stand it up on a home server or a low-cost VPS, and move it between hosts whenever you want.

Run

Local model serving

Your own model inference on localhost. Nothing leaves the box. Runs on modest hardware.

Remember

Agent memory & knowledge

A memory and knowledge store you own — with a portable format, so you can move your AI's state.

Recall

Private retrieval

Search across your own files and notes, locally. No cloud, no third-party index.

Protect

Secure tool-gateway

Every tool the agent calls is checked — schema pinning, injection scanning, secret redaction, and approval for anything dangerous.

Contain

Sandbox & egress control

The agent runs in an OS-enforced sandbox with a network kill-switch. It cannot reach out or damage the host.

Control

Admin console

Configure, monitor, and approve from one place — and export the entire stack to another machine.

The research

What doesn't exist yet.

Self-hosting individual AI parts is solved. Running a capable, tool-using agent safely and portably on your own hardware — as a turnkey stack — is not. That is what Pallium builds.

  • One policy, three boundaries. A single declarative rule spanning the tool, OS, and network layers at once.
  • Reproducible packaging. Models, memory, tools, and security as one reproducible NixOS module set.
  • Portable AI state. Export and move your memory, knowledge, and configuration between hosts — no lock-in.
  • Safe by default. The defaults that let a non-expert run an autonomous agent safely at home.
